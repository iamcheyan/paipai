# 拍拍 — 产品规格（已定稿）

**最后更新**: 2026-06-26
**状态**: 头脑风暴已定方向，进入 demo 准备

---

## 一句话

**拍拍** = 地点驱动的智能取景教练 + 拍照社交榜单。用户在网红打卡点拍照时，APP 用简单提示帮 Ta 找到最佳角度；拍完后作品挂在该地理位置上，后来者可见、可赞、可留言——像大众点评，但晒的是「在这里怎么拍出大片」。

---

## 目标用户

- 爱旅行、爱打卡的普通用户（不会构图、不会找角度）
- 想在社交媒体发高质量照片但缺摄影功底的人
- 到网红景点「不会拍」的游客

---

## 核心场景

> 用户站在某个地标/景点/餐厅门口，打开拍拍 → 选择当前位置（或地图自动识别）→ 看到前人在这个点拍过的大片 → 点击「跟拍」→ 相机界面出现取景引导（角度、站位、构图线）→ 按提示调整 → 一键拍出满意照片 → 上传到该地点榜单 → 下一位来的人继续浏览、点赞、留言。

---

## 两大核心功能

### 1. 智能取景引导（拍照教练）

| 能力 | 说明 | Demo 优先级 |
|------|------|-------------|
| 地点绑定 | 每个 POI 有预设「黄金机位」参数 | P0 |
| 取景提示 | 文字/图标提示：抬高、左移、后退、对准地标 | P0 |
| 参考叠图 | 半透明「鬼影」参考图，用户对齐即成片 | P0 |
| 角度检测 | 陀螺仪判断手机俯仰/水平是否到位 | P1（可 mock） |
| AI 构图建议 | 实时 AI 分析画面 | P2（不做，幻灯片带过） |

### 2. 地点拍照社交（榜单）

| 能力 | 说明 | Demo 优先级 |
|------|------|-------------|
| 地图选点 | Google Maps 展示 POI，点击进入地点页 | P0 |
| 地点相册 | 该位置所有用户上传的照片流 | P0 |
| 点赞 | 对单张照片点赞 | P0 |
| 留言 | 评论互动 | P1 |
| 榜单排序 | 按点赞/热度排序的「此地最佳」 | P0 |
| 用户主页 | 个人作品墙 | P2（mock） |

## 与竞品的差异

| | 拍拍 | 小红书/Instagram | 大众点评 | 原生相机 |
|---|------|------------------|----------|----------|
| 地点绑定 | ✅ 核心 | 可选标签 | 商户评价 | ❌ |
| 取景教学 | ✅ 核心 | ❌ | ❌ | ❌ |

## 用户旅程（Happy Path）

```
打开 APP
  → 地图页（Google Maps，显示附近热门拍照点）
  → 点击地点「外滩观景台」
  → 地点详情：榜单 Top 照片 + 「跟拍」按钮
  → 相机页：叠图引导 + 实时提示「向左一步」「抬高 15°」
  → 拍照 → 预览 → 上传
  → 照片出现在地点榜单，可收到点赞
```

## 技术要点（建议）

| 模块 | 方案 | 备注 |
|------|------|------|
| 前端 | React Native (Expo) 或 Flutter | 需相机 + 地图；Expo 黑客松更快 |
| 地图 | Google Maps Platform（Maps + Places） | 需 API Key，提前申请 |
| 取景引导 | 预设 JSON（机位参数 + 参考图）+ 设备 orientation | 先 2–3 个 demo 点位 |
| 社交数据 | Firebase / Supabase 或本地 mock JSON | 黑客松可先 mock |
| 图片存储 | Cloudinary / Firebase Storage 或 base64 本地 | demo 可本地 |

## Demo 点位建议（明天预选 2–3 个）

选 **有辨识度地标** + **团队能提前拍一张参考「大片」** 的地点：

1. 黑客松场地附近地标（最稳，现场可实拍）

- 1 张「黄金参考图」
- 取景提示文案（3–5 条）
- 3–5 张 mock 社区照片（`demo/mock-data/`）
