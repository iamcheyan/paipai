## 按时间段分工

### 开场 30 min（全员）

| 谁 | 做什么 |
|----|--------|
| 发言 | 确认 pitch 话术、demo 顺序 |
| 设计师 | 定主色 + 相机 overlay 线框 |
| 开发 A+B | 定技术栈（建议 Expo）、拉 `src/` 骨架 |
| 你 | 更新 `locations.json` 坐标、准备 `.env.example` |

### Sprint 1（2 h）

| 谁 | 交付物 |
|----|--------|
| 开发 A | 地图页 → 地点详情页（榜单列表） |
| 开发 B | 相机页 + 半透明参考叠图 + 提示文案 |
| 设计师 | 涩谷 Stream 门口「黄金参考图」+ 3 张样例大片 |
| 发言 | 幻灯片 1–5 页（问题 / 方案 / demo 预告） |
| 你 | AI 生成组件、接 mock 数据、写 `.gitignore` |

### Sprint 2（2 h）

| 谁 | 交付物 |
|----|--------|
| 开发 A+B | 串 happy path：选点 → 跟拍 → 拍照 → 上传 → 榜单刷新 |
| 设计师 | 全套幻灯片 + app 图标 + `demo/screenshots/` |
| 发言 | 完整彩排 2 遍，控制 3–5 分钟 |
| 你 | 录 backup 视频、填 Q&A、修文档 |

### 上台前 1 h

| 谁 | 做什么 |
|----|--------|
| 发言 | 最终讲稿 |
| 开发 | 只修 blocking bug，不新功能 |
| 设计师 | 备用截图到位 |
| 你 | 确认 API Key、网络、备用机 |

## 建议技术栈（2 开发者 + AI）

**首选：Expo (React Native)**

- `expo-camera` — 相机
- `react-native-maps` — Google 地图
- 本地 state + `demo/mock-data/*.json` — 社交数据

**你 + AI 的分工建议**：

- 你负责：把需求扔给 AI → 审代码 → 粘贴进项目 → 跑通
- AI 擅长：页面组件、样式、mock 数据接入、修 lint 错误
- 真人开发者专注：相机 overlay 对齐逻辑、地图导航、联调
